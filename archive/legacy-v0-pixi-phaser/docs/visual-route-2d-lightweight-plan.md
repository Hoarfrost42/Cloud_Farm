# 《云上小岛》轻量 2D 表现路线

## 1. 新路线结论

下一条路线改为 **轻量 2D 放置小岛**。

目标不是做高复杂度 2.5D 网格经营，也不是追求商业手游主界面完整度，而是在清爽 2D 表现下让玩家愿意上手玩几分钟，并保留现有轻松放置核心循环。

核心方向：

```text
清爽 2D 天空小岛
简单可读的地块和机器
天气反馈是主角
React HUD + Canvas 动画场景
不做真正 2.5D / isometric / 网格建造
```

## 2. 视觉目标

画面应该让玩家立刻理解：

- 这里是一座云上的小岛。
- 云可以点击。
- 雨会落到作物上。
- 作物会成长并收获。
- 机器会让小岛开始半自动运转。

优先级：

1. 清楚。
2. 轻松。
3. 有反馈。
4. 愿意点。
5. 不复杂。

不追求：

- 大型经营手游 UI 密度。
- 大量地块。
- 等距网格。
- 自由摆放。
- 多建筑系统。
- 大地图。

## 3. 推荐画面结构

### 3.1 页面外壳

```text
顶部或右侧：轻量资源 / 目标 / 升级信息
中央：2D 小岛动画舞台
底部或右下：少量主要按钮
```

不需要照搬复杂主界面：

- 不做左侧完整任务栏。
- 不做底部多 Tab 导航。
- 不做大型建筑列表。
- 不做复杂商店。

### 3.2 游戏舞台

建议重新回到更简单的 2D 空间：

```text
天空背景
上方漂浮云
中间/下方小岛
小岛上 3 块作物
小岛边缘 3 台机器
雨滴从云落向作物/机器
```

对象摆放可以用固定 2D 锚点，不需要网格化：

- `cloud-left`
- `cloud-right`
- `crop-left`
- `crop-center`
- `crop-right`
- `machine-collector`
- `machine-windmill`
- `machine-prism`

## 4. 美术策略

### 4.1 资产类型

优先使用简单 2D sprite 或程序图形：

- 云：柔软 2D 云。
- 岛：侧视或轻微俯视的 2D 小岛。
- 作物：3-4 个成长阶段。
- 机器：小水槽、风车、太阳棱镜。
- 特效：雨滴、闪光、光束、水花。

### 4.2 风格

关键词：

```text
clean
soft
bright
airy
rounded
cozy
simple 2D
light idle game
```

避免：

- 像素伪 2.5D 网格岛。
- 写实厚涂。
- 重型经营手游 UI。
- 暗黑、怪诞、赛博朋克。
- 复杂地图或建筑系统。

### 4.3 现有资产处理

当前已有像素伪 2.5D PNG 资产可以暂时保留，但不作为新路线标准。

后续处理建议：

- 可继续临时占位：云、雨滴、部分机器。
- 谨慎使用：岛底座、地块、作物，因为它们强绑定伪 2.5D 角度。
- 不继续扩展：像素 2.5D asset board。

## 5. 技术方向

当前仓库里存在两套历史：

- PixiJS v0.1 核心闭环实现。
- Phaser 网格岛实验。

新 2D 路线需要先做一次技术收敛，不要继续两套并行。

可选方案：

### 方案 A：回到 PixiJS

适合：

- 保持原 AGENTS 技术栈。
- 恢复最轻量的动画舞台。
- 利用已有 v0.1 玩法闭环。

风险：

- 需要撤掉 Phaser 实验代码。
- 需要重新处理当前 App 引用。

### 方案 B：保留 Phaser，但只做 2D 舞台

适合：

- 如果后续更想要游戏引擎式场景管理。
- Phaser 可继续处理 sprite、tween、场景和输入。

风险：

- bundle 明显更大。
- 当前项目原本不需要完整引擎能力。
- 需要更新 AGENTS/Harness 的技术栈边界。

当前建议：

```text
先不要立刻删除任何代码。
下一轮先做技术收敛决策：PixiJS 回退还是 Phaser 2D 保留。
```

## 6. 新路线最小 Demo 范围

只做：

- 一个 2D 小岛舞台。
- 1-2 朵可点击云。
- 3 块作物。
- 3 台机器的固定位置。
- 雨滴、成长、收获、升级、机器反馈。
- 右侧或顶部轻量 HUD。

不做：

- 2.5D 网格。
- 自由建造。
- 地图扩建。
- 多作物系统。
- 新资源。
- 新机器。
- 新剧情。
- 大型 UI 框架。
