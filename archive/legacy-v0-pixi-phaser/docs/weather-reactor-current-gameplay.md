# Cloud Island Weather Reactor: 当前玩法现状

更新时间：2026-05-15  
当前经济版本：`Batch 1 / economy v12 lab formula C`

## 核心定位

当前游戏已经从“种田小岛”转成一个天气主题增量游戏：

```text
点击云层注入天气活力
-> 买本轮升级提高点击和自动增长
-> 达到门槛后凝结雨阶
-> 雨阶重置本轮并提供更高天气活力倍率
-> 解锁生产者链
-> 达到 10 雨阶和季风目标后执行季风循环
-> 获得云核，购买永久升级
```

主题表达是“从第一次点击云层降下雨水开始，重塑这个空岛的天气活力”。

## 当前资源

| 资源 | 作用 | 当前问题 |
|---|---|---|
| 天气活力 | 主数值，购买大部分本轮升级，触发云层等级、雨阶和季风循环 | 增长速度跨度大，缺少高速数字滚动表现 |
| 雨滴 | 作为 `1 + log10(1 + droplets) * 50` 的天气活力乘区 | 参数仍需试玩验证，可能偏弱或偏强 |
| 根系 | 生产雨滴 | 生产者链中段 |
| 云团 | 生产根系 | 生产者链上段 |
| 雨阶 | 小 reset 等级，重置本轮并让天气活力收入变为 `1 + 雨阶` 倍 | 10 雨阶仍偏漫长 |
| 云核 | 季风循环后的永久货币 | 后续自动化和压缩旧流程 |

## 当前阶段

### 1. 开局点击

- 初始每次点击获得 `1` 天气活力。
- 基础点击冷却为 `2s`。
- 第一批升级通过云层等级自动披露，不需要额外购买“云层等级”。
- `云层注入` 用 x5 提高点击收益。
- `活力基流` 提供天气活力基础自动增长，每级增加 `1/s`。
- `天气增幅` 在 100 天气活力档提前出现，提供独立天气活力外部乘区，每级 `x4`。

### 2. 雨阶循环

- 雨阶是当前小 reset。
- 每次只能获得 `1` 雨阶，没有批量雨阶。
- 雨阶 reset 会重置当前本轮资源和本轮升级。
- 雨阶保留层级：云核、永久升级、季风循环次数、总云核。
- 雨阶倍率：所有天气活力收入乘以 `1 + 雨阶`。

### 3. 生产者链

生产者链在雨阶和云层等级推进后披露：

```text
风眼 -> 云团 -> 根系 -> 雨滴 -> 天气活力
```

当前定位：

- `根系苏醒` 打开雨滴生产。
- `云团孕育` 打开根系生产。
- `风眼牵引` 打开云团生产。
- `厚云降雨` 是较后段的天气活力和点击乘区。

### 4. 季风循环

季风循环是第一层大 reset：

- 需要 `10` 雨阶。
- 需要已经形成风眼。
- 需要天气活力达到当前季风目标。
- 第一次季风目标当前是 `1e20` 天气活力。
- 季风后重置本轮，获得云核。

### 5. 云核天赋

云核用于购买永久升级：

- 开局雨阶。
- 保留活力基流。
- 自动点击。
- 雨阶 reset 后保留第一组升级。
- 自动凝结雨阶。
- 根系生产雨滴 +25%。
- 季风额外云核。

## 当前 UI 结构

左侧：

- 版本号。
- 主资源摘要。
- 下一目标。
- 资源明细。
- 暂停、显示模式、重置存档。

中间：

- 主点击云层。
- 当前天气链条展示。
- 雨阶 reset、季风 reset、天空心脏进度。

右侧：

- 本轮升级分组。
- 云核永久天赋。

## 当前体验问题

1. formula C 刚改完，需要重新用试玩验证 10 雨阶、第一次季风的目标时长。
2. 雨滴 log 乘区解决了 formula A 过爆的问题，但可能让雨滴存在感下降。
3. 数字增长表现缺少“飞速膨胀”的视觉反馈。现在只是 React 文本刷新，没有高速滚动、增量浮字或数值动量。
4. 大数已经改为科学计数法，但卡片布局还需要继续验证。
5. 生产者链公式可玩，但根系/云团仍沿用 formula A 的存量吃乘区，后续要决定是否继续解耦。
6. 当前使用 JavaScript `number`，后期如果超过 `1e308` 会失效；目前 formula C 还没到这个范围。
