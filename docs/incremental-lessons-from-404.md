# 《云上小岛》增量玩法经验迁移说明

来源：`F:\Who_Live_in_404` 近期增量化尝试  
日期：2026-05-14

## 1. 迁移结论

《今晚谁住 404》更适合做静态空间优化 / 轻解谜，不适合强行改成传统增量游戏。它的核心乐趣来自“安排谁住哪间房”和规则结算，玩家主要在找布局最优解；如果把它改成放置增量，很容易变成手动推进、规则台账和 UI 面板堆叠。

《云上小岛》更适合承接增量 / 放置方向，因为它天然有一条清楚的增长线：

```text
云 -> 雨 -> 作物成长 -> 收获 -> 升级 -> 天气机器 -> 半自动循环
```

后续开发应把这条线做得更顺、更自动、更有即时反馈，而不是把系统数量做多。

## 2. 404 尝试暴露的问题

### 2.1 手动推进会破坏增量感

传统增量游戏的底层感受是“数值自己在走，我通过升级改变它怎么走”。如果核心推进依赖手动点“下一天 / 下一轮 / 结算”，玩家会感觉是在操作回合制原型，而不是放置游戏。

Cloud Farm 需要坚持：

- 主要资源自动增长。
- 玩家点击负责加速、触发、收获、升级。
- 阶段变化由资源、产速、建筑等级和自动链达成触发。
- 不加入手动推进天数或手动 tick 按钮。

### 2.2 空间玩法不能抢走主线

404 的空间摆放是主玩法，所以 UI 和规则解释天然复杂。Cloud Farm 可以有 5x5 岛面，但空间关系只能做增量循环的辅助表达。

Cloud Farm 的空间使用边界：

- 地块承载建筑、成长、天气影响和轻量邻接。
- 不做复杂拼图、不做最优布局求解、不做房间式标签矩阵。
- 邻接只服务第一条半自动链，例如雨水收集器 Lv.2 湿润相邻麦田。
- 地图上的对象必须一眼能读懂正在产出什么、需要什么、刚发生了什么。

### 2.3 多线系统会稀释主目标

404 的问题之一是需求清晰但维度多：房间形状、标签、租客需求、邻接、结算文案、视觉语义都要同时成立。增量游戏早期不能这样展开。

Cloud Farm v0.1 / v0.2 应坚持单一主目标：

```text
让岛从手动下雨，进入第一条半自动天气生产链。
```

所有新功能都要回答：

- 它是否提高产速？
- 它是否降低重复操作？
- 它是否让下一次升级更明确？
- 它是否能在 10 分钟内被感受到？

答不上来就延后。

## 3. Cloud Farm 的增量认知标准

### 3.1 第一屏必须让玩家知道主数值在增长

左侧资源列表可以保留，但需要有一个更高层的摘要，例如：

```text
岛屿产出：12.4 / 秒
下一目标：让雨水收集器自动湿润 1 次
```

玩家不应该只看到 6 个资源各自跳动，却不知道自己整体是在变强。

### 3.2 购买升级必须立刻改变节奏

升级不是为了填满列表，而是让玩家感到“刚才的动作改变了机器”。

优先升级类型：

- 产速提升：每秒食物 / 雨水 / 木头增加。
- 点击效率提升：一次天气施放成长更多。
- 自动化推进：周期性湿润、周期性光照、自动收集。
- 瓶颈缓解：减少某个长期卡住的资源成本。

低优先级：

- 只改说明文案。
- 只解锁另一个面板。
- 只增加一个新资源。
- 只让玩家多点一次。

### 3.3 解锁应基于循环证明

不要用“第 3 天解锁”“第 5 阶段解锁”这种外部节奏。更适合的门槛是：

- 收获 1 次麦田 -> 解锁基础升级。
- 累计获得 40 食物 -> 解锁伐木场。
- 建造雨水收集器 -> 解锁下雨局部施放目标。
- 雨水收集器 Lv.2 且相邻麦田 -> 触发自动湿润。
- 自动湿润 1 次 -> 进入半自动循环阶段。

这样玩家会理解是自己的循环能力打开了新内容。

## 4. UI 和反馈迁移规则

从 404 得到的最重要 UI 经验是：游戏对象要在主场景里表达玩法含义，面板只能解释和购买，不能替主场景承载全部意义。

Cloud Farm 应保持：

- 中央岛面显示建筑、成长、天气影响、自动触发和成熟提示。
- 左侧显示资源、总产速、当前瓶颈、下一目标。
- 右侧显示建筑和升级购买。
- 反馈优先出现在地图对象旁边，再同步到资源数字。

避免：

- 把中央地图做成纯按钮网格。
- 把成长、自动化、瓶颈都塞进侧栏文字。
- 为每个小系统新增独立面板。
- 用复杂装饰掩盖操作反馈不清。

## 5. 推荐下一批实现方向

下一轮不建议新增机器或资源。建议做“增量手感校准批次”：

1. 强化主产出摘要：显示总产出 / 秒、当前最大瓶颈、下一目标。
2. 把阶段目标改成循环证明：收获、升级、自动湿润，而不是纯数字堆叠。
3. 让第一条自动链更明显：雨水收集器 Lv.2 自动湿润相邻麦田时有地图跳字、计时或进度提示。
4. 降低认知负担：右侧升级只突出当前可买和最接近可买项。
5. 做 1 / 3 / 5 / 7 / 10 分钟手动记录，再和 `npm run simulate:ten-minute` 对照。

## 6. 禁止迁移的内容

不要从 404 搬这些东西：

- 房间模块拼图。
- 多角色规则结算。
- 邻接冲突台账。
- 大量解释性结算文本。
- 桌游式物业管理 UI。
- 怪物旅馆题材。
- 复杂美术契约。

Cloud Farm 只吸收一件事：让主循环、视觉语义和开发边界更清楚。
