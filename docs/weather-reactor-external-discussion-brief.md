# 给外部 GPT 的讨论简报

项目：Cloud Island / 云上小岛  
当前方向：天气主题增量游戏  
当前经济版本：`Batch 1 / economy v11`

## 我们想讨论什么

目标是把当前游戏从“能跑的增量原型”推进到“数值曲线清晰、前期爽、中期有平台、最终目标可期待”的小品级增量游戏。

这不是讨论 UI 细节优先级，而是讨论核心经济：

- 数值曲线怎么涨。
- 升级成本和收益怎么平衡。
- Reset 的收益和门槛怎么设计。
- 怎么避免升级公式、文案和 UI 展示互相污染。

## 当前核心循环

```text
点击云层
-> 获得天气活力
-> 买点击和被动升级
-> 凝结雨阶，重置本轮
-> 雨阶提高天气活力倍率
-> 解锁生产者链
-> 到 10 雨阶和季风目标
-> 季风循环，获得云核
-> 云核购买永久升级，压缩旧流程
```

## 当前最重要的数值

- 初始点击：`1` 天气活力。
- 点击冷却：`2s`。
- 点击升级：每级点击 `x5`。
- 雨滴生成：Lv.1 为 `1/s`，之后每级 `x4`。
- 雨阶倍率：天气活力收入 `1 + rainRanks`。
- 第一次雨阶需求：`3,000,000`。
- 第 10 雨阶需求：`1e20`。
- 第一次季风目标：`1e27`。
- 季风要求：`10` 雨阶 + 风眼。

## 当前模拟表现

自动策略大概结果：

- 第 1 雨阶：5 到 15 分钟。
- 第 3 雨阶：10 到 44 分钟。
- 第 6 雨阶：18 到 22 分钟，错误策略会卡住。
- 第 10 雨阶：约 72 分钟或更久。
- 第一次季风：约 134 到 156 分钟。

主观判断：

- 第一次雨阶时间可以接受。
- 第 10 雨阶偏长。
- 第一次季风偏远。
- 当前数值能防止玩家 3 分钟冲到高 reset，但压得有点重。

## 已修复的问题

之前升级文案用“购买后全局总收益 / 购买前全局总收益”显示倍率，导致 `雨滴生成` 明明是 x4，却显示 3.946。现在升级文案改为显示升级自身的局部倍率。

大数显示也从字母后缀改为科学计数法，避免卡片溢出。

## 仍需重点讨论的问题

1. 第 1 到第 10 雨阶的目标时长应该是多少。
2. 雨阶需求表是否应该继续使用 `3M / 10M / 30M / 100M / 1e9 / 1e11 / ...`。
3. 雨阶倍率 `1 + rainRanks` 是否太弱，还是需求表太陡。
4. 第一次季风目标 `1e27` 是否过高。
5. 生产者链应该更像 IMR 的嵌套 producer，还是更像 Shark Incremental 的清晰 10 倍门槛。
6. 云核第一次只给 1 个是否过保守。
7. 是否需要把“数字表现层”和“真实经济层”分离，比如引入 Decimal 或自定义数值封装。
8. 数字动画应该如何做，才能有经典增量游戏的高速增长感。
