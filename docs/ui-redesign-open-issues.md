# UI Redesign Open Issues

适用项目：Cloud Island / 云上小岛 / Weather Reactor  
当前分支：`ui-single-screen-hud-redesign`  
当前版本：`v13 / Post-Monsoon Complete Slice`  
最后更新：2026-05-17

## 当前状态

当前 UI 已从旧三栏调试面板过渡到经典增量工作台方向：

```text
顶部状态栏 + 主系统 tab + 左侧空岛舞台 + 右侧工作台 + 底部主线进度
```

已完成的基础修正：

- 资源、重置、图谱入口开始做阶段化显露。
- 本轮升级列表改为“所有当前组升级可见，可购买项高亮”。
- 升级按钮状态已统一：只有真正可购买/可执行时显示“购买/执行”；资源不足、未显露、已满级显示“等待”或“MAX”。
- “反应堆”tab 已改名为“概览”，降低语义误导。
- 顶部和底部略微增高，底部主线进度条加粗。
- 右侧重复阶段摘要卡片已移除。

这仍是过渡版，不是最终 UI 方案。

## 主要痛点

### 1. 首屏结构仍偏工具台

当前布局比旧三栏清晰，但仍像一个开发调试工作台：

- 左侧舞台承担了视觉，但右侧工作台仍是主要信息中心。
- 玩家仍然会先看按钮和数字，再感到“空岛天气正在变化”。
- Mood 系统目前更多是背景状态，还没有成为主体验。

期望方向：

- 空岛舞台应更像游戏主体，而不是数字面板旁的背景。
- 工作台应服务当前阶段，不应抢掉舞台的主视觉权重。

### 2. 右侧升级区存在大面积空白

一个阶段通常只有 3-5 个升级。当前右侧工作台在大屏上会出现大量空白。

已尝试方案：

- 在升级列表下方加入“当前目标 / 本组节奏 / 建议关注 / 关键资源”摘要卡片。

结论：

- 这组摘要与左侧目标、顶部资源重复，信息价值不足。
- 已删除，暂时保留空白，后续重新设计。

可能方向：

- 右侧下半区做成“当前系统图谱”的轻量可视化，而不是重复文本卡。
- 让空白成为视觉呼吸区，配合天气线、节点、阶段轨迹。
- 或改为 Overlay/Drawer，把右侧常驻工作台进一步收缩。

### 3. 顶部资源条不够理想

当前顶部资源条被改成紧凑关键读数带，但这不是最终答案。

用户明确要求：

```text
玩家永远要有一个地方能一眼观察到所有资源。
资源本身也要渐进式披露。
顶部关键资源摘要不能替代完整资源观察区。
```

当前问题：

- 顶部如果塞所有资源，后期会装不下，也不好看。
- 顶部如果只放关键资源，玩家缺少“一眼看全资源”的位置。
- “资源”tab 可以显示完整资源，但 tab 内查看不等于常驻一眼可见。

待设计问题：

- 完整资源观察区应该放在顶部、底部、右侧、抽屉，还是可展开资源带？
- 后期资源很多时，如何分层显示而不变成拥挤仪表盘？
- 资源显露顺序要和玩法阶段一致，避免开局暴露云核、气压、风暴胞、气候织线。

### 4. 顶部资源与 tab 区分度不足

当前顶部资源区、主 tab、右侧工作台都使用相近的浅色卡片语言。

表现：

- 资源读数容易被玩家忽略。
- 顶部状态栏与 tab 条视觉层级接近。
- 资源本应是“游戏状态”，但现在像普通导航旁的小卡片。

可能方向：

- 资源带使用更明确的数字仪表语义。
- 主 tab 降低装饰，资源带提升对比。
- 给资源增加统一图标/材质，而不是普通卡片边框。

### 5. 重置系统需要继续渐进式披露

当前重置页已按阶段隐藏后期 reset，但仍需要继续检查后期状态：

- 开局/雨阶前：只显示凝结雨阶。
- 接近 10 雨阶或可季风：显示季风循环。
- 第一次季风后：显示前线回响/风暴前线。
- 第一风暴后：显示气候改写。
- 天空心脏阶段再显示天空心脏脉冲和终局。

风险：

- 后期某些 reset 可能过早暴露。
- reset 文案可能仍偏技术说明。
- “前线回响”等补偿 reset 需要更清楚表达它和主 reset 的关系。

### 6. “概览”页意义仍弱

“反应堆”已改成“概览”，但内容仍是临时摘要。

当前风险：

- 玩家可能不明白为什么要点概览。
- 概览页如果只重复目标和推荐升级，会继续制造冗余。

可能方向：

- 概览页只在后期承担“当前系统关系图”。
- 开局可以默认进入“本轮升级”，概览后续再显露。
- 概览页可成为“本阶段收益来源 / 主要瓶颈 / 下一个系统”的可视化面板。

### 7. 右侧升级行长度仍偏长

已通过 `max-width` 控制右侧行宽，但在宽屏下仍显得横向很长。

当前表现：

- 行内文字与按钮相隔远。
- 右侧工作台像表格，而不是游戏升级区。

可能方向：

- 升级列表改为固定中等宽度，右侧留给阶段图谱或说明。
- 升级行改为更紧凑的两列布局。
- 购买按钮与价格合并成更强的动作区。

### 8. 空岛舞台的成就反馈不足

用户希望加入 Mood/生态反馈：

```text
玩家进度越高，空岛天气和生态越丰富，有实际成就感。
```

当前问题：

- 舞台有天气线、云层、背景色，但反馈仍偏抽象。
- 雨阶、季风、风暴、气候改写对场景的影响还不够直观。
- 玩家长期游玩后的“岛变好了”感不足。

可能方向：

- 雨阶提升后增加雨线密度、水汽、浅绿。
- 根系阶段出现岛面纹理或根脉。
- 风眼阶段出现环流线。
- 季风后出现云核记忆/厚云层。
- 风暴后出现前线边缘和更强天气动势。
- 气候改写后出现规则纹理或气候环。
- 天空心脏阶段出现高空脉冲。

### 9. 后期系统入口需要更清楚

后期包括：

- 云核天赋
- 气压升级
- 风暴图谱
- 气候法则
- 天空心脏

当前问题：

- 这些系统现在主要通过 tab 或图谱页进入。
- 解锁时缺少强提示和状态过渡。
- 玩家可能不知道“我现在进入了新循环”。

需要设计：

- 新系统解锁时如何提示。
- 新系统第一次出现时是否自动切换/高亮入口。
- 后期资源和永久升级如何在 UI 上建立层级关系。

### 10. UI 语言还不够统一

当前 UI 混合了：

- 经典增量游戏表格/按钮语言。
- 空岛天气舞台语言。
- SaaS 式卡片/面板语言。

风险：

- 看起来比旧版清楚，但仍不像最终游戏。
- 卡片边框、浅色面板、按钮样式过多，视觉语义重复。

需要统一：

- 哪些是资源。
- 哪些是可操作升级。
- 哪些是阶段/目标。
- 哪些是系统入口。
- 哪些只是背景或氛围。

## 当前代码层注意事项

### 已修状态

- `UpgradeRow` 已统一 action label：
  - 可购买：`购买`
  - 不可购买：`等待`
  - 已拥有：`已拥有`
  - 满级：`MAX`
- `WorkbenchPanel` 中本轮升级、云核天赋、气压/风暴/气候升级均使用同一套 disabled 语义。
- `getUnlockedMainTabs` 中 `reactor` 的显示名称已从“反应堆”改为“概览”。

### 仍需小心

- 顶部资源目前只显示 `getVisibleHudStats(...).slice(0, 4)`，这是临时状态，不代表最终资源方案。
- 完整资源观察区必须重新设计。
- 右侧空白问题暂时没有最终方案。
- 当前 UI 分支没有改数值逻辑，后续 UI 改动应继续避免影响模拟结果。

## 建议下一步

优先让 GPT Pro / 设计讨论聚焦三个问题：

1. **资源观察方案**
   - 顶部关键摘要和完整资源一眼观察区如何共存？
   - 后期资源多时如何渐进披露？

2. **右侧空白方案**
   - 空白是保留呼吸、做阶段图谱、做系统轨迹，还是进一步减少右侧常驻面积？

3. **空岛 Mood 视觉方案**
   - 如何让进度变化在舞台上被感知，而不是只体现在数字和按钮上？

验收标准应保持：

- 首屏无整页滚动条。
- 开局不暴露后期资源。
- 玩家 3 秒内知道当前阶段、关键资源、下一步动作。
- 所有资源有一个可一眼观察的地方。
- 完整资源和公式不挤占主舞台。
